# Claude Code에게 전달할 프롬프트

아래 프롬프트를 Claude Code 세션에 그대로 복사해서 붙여 넣으세요. 핸드오프 폴더(`design_handoff_a11ycert_redesign/`)는 운영 레포지토리 루트에 함께 두거나 첨부해서 함께 보여주면 됩니다.

---

## 시작 프롬프트 (복사해서 사용)

```
A11yCert (www.a11ycert.com) 웹사이트의 전면 리디자인을 운영 코드베이스에 적용하려고 합니다.

레퍼런스 자료가 `design_handoff_a11ycert_redesign/` 폴더에 있습니다. 우선 다음 순서로 작업해 주세요.

1) `design_handoff_a11ycert_redesign/README.md`를 처음부터 끝까지 정독하고, 디자인 의도와 컴포넌트 구조를 요약해서 알려 주세요.
2) `design_handoff_a11ycert_redesign/reference/` 안의 HTML/CSS/JS를 모두 읽고, 어떤 토큰·컴포넌트·레이아웃을 운영 코드로 옮겨야 하는지 매핑 표를 만들어 주세요.
3) `design_handoff_a11ycert_redesign/screenshots/` 안의 PNG들을 한 페이지씩 보고, 라이트/다크/고대비 3 테마가 어떻게 다른지 확인해 주세요.
4) 운영 코드베이스의 현재 구조(Next.js 버전, i18n 라이브러리, 스타일링 방식, 컴포넌트 라이브러리)를 파악한 뒤, 작업 계획을 단계별로 제안해 주세요. 계획에는 반드시 다음을 포함:
   - 디자인 토큰을 어디에 어떤 형태로 둘지 (예: globals.css의 CSS 변수 / Tailwind config)
   - 테마 컨텍스트(라이트/다크/고대비) 구현 위치와 SSR 깜빡임 방지 방법
   - 페이지 7개를 어떤 라우트와 어떤 컴포넌트로 구성할지
   - localStorage 기반 상태 훅(progress / bookmarks / wrongNotes / flashSchedule)의 위치
   - i18n 키 구조 (한국어 카피 + 영어 원문 동시 표시 방식)
5) 계획에 OK 사인을 받기 전에는 코드를 변경하지 마세요. 계획 단계에서 멈춰 주세요.

작업 원칙:
- HTML 레퍼런스를 그대로 복붙하지 말고, 운영 코드의 기존 컨벤션(컴포넌트 분할, 스타일링 라이브러리, 상태 관리)에 맞춰 재구현합니다.
- 디자인 토큰 값(색상, 간격, 타이포)은 README의 표를 그대로 사용해 픽셀 정확도를 유지합니다.
- 모든 인터랙티브 요소는 키보드 접근 가능해야 하고, `:focus-visible` 링이 보여야 합니다.
- 라이트/다크/고대비 모든 모드에서 WCAG AA 이상 대비를 유지합니다 (고대비는 AAA).
- prefers-reduced-motion이면 모든 transition/animation을 0.01ms로 줄입니다.
- 기존 사용자의 학습 진행도가 있다면 1회 마이그레이션 스크립트를 만들어 신 키로 옮깁니다.

먼저 1번부터 시작해 주세요.
```


## 진행 권장 순서 (체크리스트)

### Phase 1 — 기반 (1~2일)
- [ ] 디자인 토큰 이식 (`tokens.css` → 운영 코드의 글로벌 CSS 또는 Tailwind config)
- [ ] 테마 프로바이더 구현 (라이트/다크/고대비 + SSR 깜빡임 차단 스크립트)
- [ ] 폰트 로딩 (Pretendard Variable, Noto Serif KR, JetBrains Mono)
- [ ] 글자 크기 4단계 + 모션 줄이기 토글

### Phase 2 — 셸 (1일)
- [ ] `<AppHeader>` (데스크톱 7-item 네비 + 언어 토글 + 모드 메뉴)
- [ ] `<MobileSheet>` (햄버거 → 바텀시트, 980px↓에서 활성)
- [ ] `<AppFooter>` (4열 → 모바일 2열 → 1열)
- [ ] 본문 바로가기 링크

### Phase 3 — 정적 페이지 (2~3일)
- [ ] 홈 (`/ko`, `/en`)
- [ ] CPACC 개요 (`/ko/cpacc`)
- [ ] WAS 개요 (`/ko/was`)
- [ ] 용어집 (`/ko/glossary`) — 검색 + 필터 동작 포함

### Phase 4 — 학습 도구 (3~5일)
- [ ] 학습 페이지 — 사이드바 + 이중언어 카드 + 키보드 단축키
- [ ] 퀴즈 — 즉시 채점 + 해설 + 자동 오답 노트 (localStorage)
- [ ] 플래시카드 — 3D flip + SM-2 기반 간격 반복 스케줄링

### Phase 5 — 마이그레이션 & QA (1~2일)
- [ ] 기존 사용자의 진행도/북마크 1회 마이그레이션
- [ ] axe-core / Lighthouse 접근성 검증
- [ ] 키보드 전용 시나리오 1바퀴
- [ ] 스크린 리더 (NVDA + VoiceOver) 1바퀴
- [ ] 라이트/다크/고대비 × 모바일/데스크톱 시각 회귀

## 주의사항

- **HTML 그대로 옮기지 않기**. 레퍼런스의 마크업/스타일은 디자인 의도를 보여주기 위한 것이고, 운영 코드는 컴포넌트로 분리해 재사용 가능하게 만들어야 합니다.
- **로직 분리**. 퀴즈/플래시카드 알고리즘(SM-2)은 순수 함수로 분리해 단위 테스트 가능하게 만들어 주세요.
- **i18n 분리**. 한국어 카피와 영어 원문은 i18n 리소스로 분리하고, 학습 카드의 한/영 동시 표시는 두 키를 동시에 가져와 렌더하는 식으로 구현합니다.
- **퍼포먼스**. 폰트는 `font-display: swap`, 다크/고대비 모드 전환 시 깜빡임 없이 적용되도록 차단형 스크립트 사용.
